# 示例：Product Judge 商业评估节选

## 输入

```text
帮我用 vivi-biz-review 测评一下我 GitHub 里面 Product Judge 的项目。
```

## 输出节选

**是否值得继续**：值得做，但先想清楚再做

**证据等级**：中

**一句话理由**：方向有真实需求，竞品也证明有人愿意为“想法验证报告”付费；但 Product Judge 现在同时想做对话、专家点评、知识库、灵感卡片、报告导出，主购买理由还不够尖。

## 核心判断

Product Judge 的方向是对的，但定位应该收窄。

“AI 产品顾问”太宽，用户第一次进来时不知道自己到底要买什么。

更尖的定位是：

```text
中文独立开发者的产品想法判官。
```

它不应该只是生成报告，而应该帮用户在动手开发前回答：

- 这个用户真的存在吗？
- 他现在怎么解决？
- 为什么会换成我的方案？
- 这个方向有没有可能变成一门小生意？
- 这周最该验证什么？
