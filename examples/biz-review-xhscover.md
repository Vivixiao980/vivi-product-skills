# 示例：xhs-cover-skill 商业评估节选

## 输入

```text
帮我用 vivi-biz-review 跑一下 xhs-cover-skill 这个项目。
```

## 输出节选

**是否值得继续**：值得做，但先想清楚再做

**证据等级**：中

**一句话理由**：小红书封面需求真实，但 CLI/Skill 形态太窄；真正商业机会不在“开源 Skill 收费”，而在“创作者可直接用的封面生成器 + 风格库 + 内容工作流”。

## 核心判断

xhs-cover-skill 技术完成度不错，已经支持：

- Claude Code / OpenClaw Skill 调用
- 18 种预设风格
- Gemini 图片生成
- 人像照片 + 标题合成封面
- 本地配置和输出

但它的目标用户有两层：

- 小红书创作者真的需要封面。
- 会安装 Claude Code Skill 的人很少。

所以更好的商业路径不是卖一个 Skill，而是把它变成：

```text
普通创作者能直接使用的网页版封面工具，
或者小红书内容自动化套件。
```
